还有个设定，如何让跑马灯首位可以相连？目前是整个字段放映结束之后才开始第二遍放映，中间有段很长的空白

由于原生的 CSS 特效中，单个标签在位移时无法凭空“克隆”自己，所以常规的 animation 滚动到末尾时，必须等整个文本完全移出可视区，才会重新从右侧跳出来，这就导致了你看到的很长的空白。

要实现通俗意义上的无缝首尾相连（无缝滚动），最稳妥、优雅且不需要大改 JSX 结构的纯 CSS 办法是：利用 background-repeat 的平铺特性，把文字转为背景，或者直接通过复制一份文本进行双列拼接。

不过针对你目前的告警列表（直接在 <td> 里面包裹单行文字），最直接、对样式破坏最小的首尾相连两全之策是以下两步修改：

第一步：修改 JSX 中的包裹结构（复制一份文本）
找到 VideoAlarmManage 组件内表格 <tbody> 里的 alarm.time 和 alarm.source 这两个单元格（<td>），让内部的文字“复制一份”拼在后面。

将对应的 <td> 内部结构修改为：

JavaScript
<td className="border-b border-[var(--color-panel-border)] px-[var(--layout-search-padding-x)] py-[var(--layout-device-table-padding-y)]">
  <div className="viot-scroll-container w-full" title={alarm.time}>
    <div className="viot-scroll-inner">
      <span>{alarm.time}</span>
      {/* 复制一份接在后面，用来填补空白实现首尾相连 */}
      <span className="viot-scroll-replica">{alarm.time}</span>
    </div>
  </div>
</td>
(同样地，把紧随其后的 alarm.source 那一列也照着这个结构修改，复制一份 <span className="viot-scroll-replica">{alarm.source}</span>)

第二步：修改 CSS 动画和样式
找到组件最上方的 <style> 标签，将原本的跑马灯样式完全替换为以下更加凝练的无缝循环位移逻辑：

JavaScript
<style>{`
  @keyframes viot-auto-scroll {
    0% { transform: translateX(0); }
    100% { transform: translateX(-50%); } /* 只需要左移整个内部容器的一半（即第一份文本的宽度） */
  }
  .viot-scroll-container {
    display: block;
    width: 100%;
    overflow: hidden;
    white-space: nowrap;
  }
  .viot-scroll-inner {
    display: inline-block;
    animation: viot-auto-scroll 20s linear infinite; /* 20s 也是你刚刚调整好的慢速 */
  }
  /* 调整副本与原本之间的间距，让首尾衔接更自然 */
  .viot-scroll-replica {
    padding-left: 2rem; /* 两个文本之间的间距，可根据需要微调 */
  }
`}</style>
为什么这样能消除空白并首尾相连：
原本的动画是从 100%（右侧屏幕外）走到 -100%（左侧屏幕外）。现在由于后面紧跟了一份一模一样的副本文本，当第一份文本刚好完全向左滑出边界时，第二份文本刚好补齐在它的位置上。此时动画悄悄咪咪地瞬间重置回 0，视觉上就会变成完全没有任何断层的无限无缝首尾相连。
