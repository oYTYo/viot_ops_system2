1. 这个项目是 Vite + React + Tailwind + JSX。后续开发尽量复用现有母版，不要重新造整体布局。功能页面主要开发右侧内容区和右侧底部工具区，不要重复实现左侧行政区树、顶部栏、底部栏、主题切换、字号切换这些母版能力。

2. 样式不要硬编码。字号、高度、宽度、间距、圆角、图标大小、颜色都优先使用 `src/index.css` 里的 CSS 变量和已有类名。常用字号是 `text-app-title`、`text-ui-large`、`text-ui-medium`、`text-ui-small`。不要随便写固定像素，比如 `text-[18px]`、`h-[40px]`、`w-[320px]`。优先使用类似 `text-ui-medium`、`h-[var(--layout-segment-button-height)]`、`w-[var(--layout-sidebar-width)]`、`text-[var(--color-text-main)]`、`bg-[var(--color-panel-bg)]`、`border-[var(--color-panel-border)]` 这种写法。这样才能保证大字/小字模式、浅色/深色模式都能自适应。

3. 当前母版结构大致是：`TopBar` 负责顶部功能导航，`Sidebar` 负责左侧行政区和摄像机树，右侧内容区负责当前功能页面，`BottomBar` 负责底部工具栏。后续功能开发不要在每个页面里重新写行政区树。左侧树是全局共享业务目录，结构是省级、地级、县级、乡级、摄像机。

4. 左侧行政区树不是普通菜单，而是设备目录。行政区后面的数字会随筛选变化。选择“全部”时显示在线设备数/全部设备数；选择“正常”时显示正常设备数/全部设备数；选择“异常”时显示异常设备数/全部设备数；选择“离线”时显示离线设备数/全部设备数。单个摄像机后面不显示 `(1/1)`，只显示状态块，比如正常、异常、离线。

5. 摄像机状态字段不要乱改。数据库和后端里的摄像机状态仍然是 `online`、`fault`、`offline`。前端下拉框显示的是“全部 / 正常 / 异常 / 离线”。其中“正常”对应 `status === "online"`，“异常”对应 `status === "fault"`，“离线”对应 `status === "offline"`。不要把数据库里的 `online` 改成 `normal`。

6. 摄像机图标和行政区图标要区分。行政区用文件夹图标，摄像机用摄像机图标。正常在线摄像机用高亮色，异常摄像机用 error 色，离线摄像机用灰色。左侧树的展开状态要保留。用户切换“全部 / 正常 / 异常 / 离线”筛选时，已经展开的目录不能关闭。展开状态由 `expandedNodeIds` 在母版层统一维护，不要在单个 `TreeNode` 里面再用本地 `open/setOpen` 管理。只有点击底部重置按钮，才恢复初始展开状态。

7. 左侧导航栏默认宽度来自 `index.css` 里的 `--layout-sidebar-width`，它和字号模式绑定，不能硬编码成 `320px`。手动拖拽后才临时使用 px 宽度；点击底部重置后，要恢复成 `var(--layout-sidebar-width)`。

8. 底部工具栏视觉上保持一整条，但逻辑上分成两块。左侧区域服务行政区树，宽度跟随 `Sidebar`；右侧区域服务当前功能页面，随顶部功能切换。后续每个功能页自己的工具按钮应该放在右侧底部工具区，不要影响左侧行政区工具。

9. 左侧树相关接口优先复用后端已有接口：`GET /nav-tree/children`、`GET /nav-tree/cameras`、`GET /nav-tree/search`、`GET /nav-tree/node`。不要让前端自己拼复杂行政区树，树结构和统计尽量让后端返回。

10. 后续每个功能页主要关心当前功能类型、当前选中的行政区或摄像机、当前筛选状态。不要重复实现行政区加载、摄像机目录、收藏夹、展开状态、筛选状态、左侧栏拖拽、主题切换、字号切换等母版能力。

11. 开发新功能时，先判断这个功能是页面自己的，还是母版共享的。如果是页面自己的，放右侧内容区或右侧底部工具区；如果是全局共享的，不要在具体页面里重复写。
